Lab 2 Report: ROS 2 CLI and RQT Interaction

In this lab, I utilized the ROS 2 command-line interface and the rqt GUI framework to interact with the turtlesim node. Initially, I used terminal commands to publish geometry_msgs/msg/Twist messages to the /turtle1/cmd_vel topic, dictating the kinematics of the default turtle. Transitioning to the rqt interface, I accessed the Service Caller plugin to execute the /spawn service, instantiating a second turtle (turtle2) at specific coordinate constraints. I then used the Message Publisher plugin in rqt to independently control the second turtle via the /turtle2/cmd_vel topic, successfully demonstrating the asynchronous nature of ROS 2 topic communication. Finally, I tested state management by triggering the /reset service via the GUI to return the simulation to its initial parameters.
